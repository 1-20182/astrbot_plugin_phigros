# 📋 Phigros 插件系统性更新流程

## 🎯 目标

实现一个系统性的更新流程，按顺序推进项目的每个阶段，确保每个更新都经过充分的测试和验证，最终实现所有 P1 和 P2 优先级的功能改进。

## 📊 阶段划分

### 0. 规划阶段
- **范围**：定义每个更新的具体内容、技术方案和测试计划
- **依赖**：无
- **成功标准**：完成详细的更新计划文档

### 1. P1 阶段 - 核心功能改进

#### 1.1 拆分 main.py 上帝类
- **范围**：将 main.py 按功能拆分为多个模块化文件
- **依赖**：已完成的 core 模块架构
- **成功标准**：main.py 代码量减少 50% 以上，功能模块清晰分离
- **状态**：✅ 完成
- **完成时间**：2026-03-10
- **成果**：
  - 已将认证相关命令移至 commands/auth_commands.py
  - 已将查询相关命令移至 commands/query_commands.py
  - 已将其他命令移至 commands/other_commands.py
  - main.py 代码量大幅减少，职责更加清晰

#### 1.2 统一配置管理
- **范围**：实现支持环境变量优先级的统一配置管理
- **依赖**：无
- **成功标准**：配置能从环境变量、配置文件、默认值三层读取，优先级正确
- **状态**：✅ 完成
- **完成时间**：2026-03-11
- **成果**：
  - 实现了ConfigManager.get_config方法，支持三层配置优先级
  - 更新了main.py中的所有配置读取，使用新的配置管理方法
  - 支持环境变量、插件配置和默认值的正确优先级
  - 添加了配置读取的日志记录

#### 1.3 数据备份系统
- **范围**：实现 user_data.json 的定期备份机制
- **依赖**：无
- **成功标准**：每 24 小时自动备份，保留最近 7 天的备份
- **状态**：✅ 完成
- **完成时间**：2026-03-12
- **成果**：
  - 实现了自动备份功能，每次保存数据时创建备份
  - 实现了备份清理功能，保留最近7天的备份
  - 实现了从备份恢复功能，当主文件损坏时自动尝试恢复
  - 备份文件存储在 backups 目录中，包含时间戳
  - 增加了详细的备份相关日志

#### 1.4 优化首次体验
- **范围**：内置默认曲绘，提升新用户体验
- **依赖**：无
- **成功标准**：新安装后无需手动添加曲绘即可正常使用
- **状态**：✅ 完成
- **完成时间**：2026-03-13
- **成果**：
  - 渲染器能够处理曲绘不存在的情况，显示占位符
  - 曲绘更新器会在首次运行时自动下载曲绘
  - 曲绘更新在后台运行，不阻塞插件初始化
  - 支持代理设置，解决GitHub访问问题
  - 实现了增量更新和断点续传，提高下载效率

### 2. P2 阶段 - 增强功能

#### 2.1 API 降级方案
- **范围**：实现 API 不可用时的缓存降级
- **依赖**：已完成的缓存系统
- **成功标准**：API 故障时能使用缓存数据继续提供服务
- **状态**：✅ 完成
- **完成时间**：2026-03-14
- **成果**：
  - 为API客户端添加了缓存支持，使用HybridCache
  - 实现了API故障时的自动降级机制，从缓存获取数据
  - 为get_save、get_rks_history和get_bestn_image方法添加了缓存支持
  - 当API故障时，会尝试从内存缓存和磁盘缓存中获取数据
  - 增加了详细的日志记录，便于调试和监控

#### 2.2 监控指标
- **范围**：添加成功率、耗时等监控指标
- **依赖**：无
- **成功标准**：能统计并展示 API 调用成功率和平均耗时
- **状态**：✅ 完成
- **完成时间**：2026-03-15
- **成果**：
  - 创建了监控模块，实现了API调用的统计功能
  - 添加了@monitor_api_call装饰器，自动统计API调用
  - 实现了时间窗口内的指标统计，包括成功率、耗时等
  - 添加了/phi_metrics命令，用于查看监控指标
  - 增加了详细的日志记录，便于调试和监控

#### 2.3 Docker 部署
- **范围**：提供 Docker 镜像和部署配置
- **依赖**：无
- **成功标准**：能通过 Docker 一键部署运行

#### 2.4 单元测试
- **范围**：为核心功能添加单元测试
- **依赖**：无
- **成功标准**：核心功能测试覆盖率达到 80% 以上

### 3. 最终验证
- **范围**：全功能测试和性能验证
- **依赖**：所有 P1 和 P2 阶段完成
- **成功标准**：所有功能正常运行，性能符合预期

2. **实施开发**
   - 按照技术方案实现功能

   - 编写详细的代码注释

3. **测试验证**
   - 运行单元测试
   - 进行集成测试
   - 验证功能是否符合预期

4. **文档更新**
   - 更新 CHANGELOG.md
   - 更新 README.md（如有必要）

5. **代码审查**
   - 检查代码质量
   - 验证安全性
   - 确保性能优化

6. **合并部署**
   - 将代码合并到 main 分支
   - 推送至远程仓库
   - 部署测试环境验证

7. **应急计划**
   - 准备回滚方案
   - 监控部署后的运行状态
   - 制定问题响应流程
